Of course, there is not the only way!

configure postgres to allow remote connection

If you want to remote connect the postgres, you need to check the process first

Then make sure the iptables allow remote connect via specified port like 5432(can chect that on the postgresql.conf file). Add security rule if you use AlibabaCloud server.

change the listen address from localhost to *

edit the pg_hba.conf to allow all/specified ip connection.

Then restart postgres

Then you can probably connect postgres with remote terminal or other tools.
